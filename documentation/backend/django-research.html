Django Research

Rationale

We decided on Django as our backend framework for the following reasons:

  • Most of our group members have experience developing in Python, so we thought a python based backend would be the best choice, rather than choosing a language that not everyone is familiar with, like TypeScript or Java.
  • Django provides documentation and easy integration to deploy the app to Heroku. Since the course requires us to use Heroku for deployment, we thought Django would be a good fit.
  • Python provides an easy interface for web sockets, which we plan to utilize in order to add voice calling to our app.
