Node.JS Research

Rationale

It is way more suited for applications that require a persistent connection between the browser and server.

Since Node.JS allows for event based (asynchronous calls) using JavaScript, it's very good for when you need to do several different things at the same time (voice chat, games, etc.)

It does not require the server to open multiple threads (like Django would have to). If we were to do this in Django, each seperate thread (user that's connected to voice call) would require its own server process. That's too much!

Instead, Node utilizes long polling, where updates can be sent directly to the user in real time. We are building an application that will need voice chat and real time text chat, so these features of Node will fit our needs the best.

Source

Since most of the teammates have React experience and we agreed to use React for the frontend, it is also nice to use a JavaScript based backend. This way, we can also use the same testing library.

Finally, there are already a LOT of examples and tutorials out there about making real-time voice chat and text chat apps using node. There's honestly way more than doing it in Django.
